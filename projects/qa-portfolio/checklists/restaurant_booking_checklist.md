# Чек-лист: Restaurant Booking App

## Навигация

- [ ] Главная страница открывается без 500-ошибки.
- [ ] Меню доступно из навигации.
- [ ] Галерея доступна из навигации.
- [ ] Контакты доступны из навигации.
- [ ] Бронирование доступно из навигации.
- [ ] Активные ссылки не ведут на несуществующие маршруты.

## Форма бронирования

- [ ] Нельзя отправить пустую форму.
- [ ] Поле имени принимает кириллицу и латиницу.
- [ ] Телефон проверяется на минимальную длину.
- [ ] Email проверяется на корректный формат.
- [ ] Дата в прошлом не принимается.
- [ ] Количество гостей не может быть меньше 1.
- [ ] После успешного бронирования пользователь видит подтверждение.

## Админка

- [ ] Список броней отображается.
- [ ] Статус брони можно изменить.
- [ ] Ошибки БД не показывают пользователю внутренний traceback.

## Безопасность

- [ ] Секретный ключ не хранится в репозитории.
- [ ] Пароль БД берётся из `.env`.
- [ ] SQL-запросы используют параметры, а не конкатенацию строк.
