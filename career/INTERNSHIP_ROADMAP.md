# План подготовки к IT-стажировке

Цель: выйти на уровень, достаточный для откликов на Python Intern, Data Analyst Intern, Junior QA / QA Automation Intern и технические стажировки.

## Этап 1. Привести портфолио в рабочий вид — 3–5 дней

1. Выложить 3–5 лучших проектов на GitHub.
2. Проверить, что каждый проект запускается по инструкции из README.
3. Добавить скриншоты графиков и результатов.
4. Убрать мусор: `venv`, `bin`, `obj`, `.exe`, `.dll`, локальные пути, пароли.
5. Добавить ссылки на GitHub в резюме.

## Этап 2. Python Core — 2 недели

Нужно уверенно знать:

- типы данных: list, tuple, dict, set, str;
- циклы, условия, функции;
- list/dict comprehensions;
- работа с файлами;
- исключения;
- модули и пакеты;
- `argparse`;
- основы ООП: class, object, inheritance, dataclass;
- виртуальные окружения и `pip`.

Практика:

- переписать старые учебные скрипты в функции;
- добавить CLI в 2 проекта;
- написать 5–10 маленьких задач на Codewars/LeetCode/Yandex Contest.

## Этап 3. Git и GitHub — 2–3 дня

Нужно знать:

- `git init`, `clone`, `status`, `add`, `commit`, `push`, `pull`;
- ветки: `branch`, `checkout/switch`, `merge`;
- `.gitignore`;
- как читать diff;
- как оформить README.

Практика:

- сделать минимум 10 осмысленных коммитов в портфолио;
- не загружать проект одним коммитом `final version`.

## Этап 4. SQL — 1–2 недели

Нужно знать:

- `SELECT`, `WHERE`, `ORDER BY`, `LIMIT`;
- `JOIN`: INNER, LEFT;
- `GROUP BY`, `HAVING`;
- агрегаты: COUNT, SUM, AVG, MIN, MAX;
- подзапросы;
- первичный/внешний ключ;
- базовая нормализация;
- индексы на уровне понимания.

Практика:

- доработать `sql-airline-queries`;
- сделать SQLite-версию с `schema.sql` и `seed.sql`;
- решить 30–50 SQL-задач на тренажёрах.

## Этап 5. Алгоритмы и структуры данных — 3–4 недели

Минимум для стажировки:

- массивы, строки, хеш-таблицы;
- стек, очередь, deque;
- сортировки и бинарный поиск;
- рекурсия;
- графы: BFS, DFS, Dijkstra/A* на базовом уровне;
- оценка сложности Big O.

Практика:

- довести `maze-pathfinding-python` до хорошего состояния;
- решить 50 задач уровня easy/medium;
- уметь объяснять сложность решений.

## Этап 6. Web/API — 1–2 недели

Нужно знать:

- HTTP: GET, POST, status codes;
- JSON;
- REST на базовом уровне;
- Flask routes;
- формы и валидация;
- `.env` и конфигурация;
- базовая безопасность: не хранить пароли в коде.

Практика:

- довести `restaurant-booking-flask`;
- добавить `schema.sql`, `seed.sql`;
- добавить 2–3 API endpoint;
- проверить через Postman.

## Этап 7. QA-база — 1 неделя

Даже для Python Intern полезно знать:

- чек-лист;
- тест-кейс;
- баг-репорт;
- severity/priority;
- smoke/regression testing;
- DevTools;
- Postman;
- основы pytest/unittest.

Практика:

- заполнить `qa-portfolio` реальными проверками своего Flask-приложения;
- написать 5 unit-тестов для Python-проектов.

## Этап 8. Подготовка к собеседованию — постоянно

Подготовь ответы:

1. Расскажи о себе за 60 секунд.
2. Почему хочешь стажировку именно в IT?
3. Какой проект самый сильный?
4. Какие трудности были в проекте SEM/lithography?
5. Почему NILS нельзя считать по SEM-изображению?
6. Чем BFS отличается от DFS?
7. Что такое JOIN?
8. Что такое HTTP status code 404/500?
9. Что такое виртуальное окружение Python?
10. Как ты проверяешь, что код работает?

## Рекомендуемая стратегия откликов

Каждую неделю:

- 20 откликов на стажировки;
- 10 откликов на junior/trainee QA или data analyst;
- 5 откликов на Python/automation intern;
- 2–3 доработки портфолио по обратной связи.

Главная цель — получить первые интервью, а не идеально подготовиться «когда-нибудь».
